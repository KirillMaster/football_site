```
                        [Главная /]
```

### 3.2. Sitemap.xml

Динамический sitemap, автоматически обновляемый при публикации контента. Генерируется Next.js через `app/sitemap.ts`.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://fcarsenal92.ru/</loc>
    <lastmod>2026-04-12</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
    <xhtml:link rel="alternate" hreflang="ru" href="https://fcarsenal92.ru/"/>
    <xhtml:link rel="alternate" hreflang="en" href="https://fcarsenal92.ru/en/"/>
  </url>
  <url>
    <loc>https://fcarsenal92.ru/about</loc>
    <lastmod>2026-04-10</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
    <xhtml:link rel="alternate" hreflang="ru" href="https://fcarsenal92.ru/about"/>
    <xhtml:link rel="alternate" hreflang="en" href="https://fcarsenal92.ru/en/about"/>
  </url>
  <!-- ... все публичные страницы, включая каждую новость и товар -->
</urlset>
```

**Правила sitemap:**
- Включать только канонические URL (HTTPS, без www)
- Включать `lastmod` для страниц с обновляемым контентом
- Не включать: /admin/*, страницы с `noindex`, 404/500
- При > 50,000 URL -- разделить на sitemap index

### 3.4. Canonical URL стратегия

- Каждая страница содержит `<link rel="canonical" href="https://fcarsenal92.ru/[path]" />`
- Канонический URL всегда: HTTPS, без www, без завершающего слеша
- Пагинация новостей: `?page=2` -- canonical указывает на текущую страницу (не на первую)
- Фильтрация магазина: `?category=boots` -- canonical указывает на URL без фильтров, если контент дублируется; или на текущий URL, если контент уникален
- Английская версия: canonical = URL английской версии (НЕ русской), hreflang связывает обе версии

### 3.5. Hreflang реализация

На каждой странице в `<head>`:

```html
<link rel="alternate" hreflang="ru" href="https://fcarsenal92.ru/about" />
<link rel="alternate" hreflang="en" href="https://fcarsenal92.ru/en/about" />
<link rel="alternate" hreflang="x-default" href="https://fcarsenal92.ru/about" />
```

**Правила:**
- `x-default` всегда указывает на русскую версию (основной язык)
- Hreflang должен быть двунаправленным (ru-страница ссылается на en, en-страница ссылается на ru)
- Также дублировать hreflang-аннотации в sitemap.xml (см. раздел 3.2)
- Реализация в Next.js: через `next-intl` middleware + `generateMetadata()` в layout.tsx

## 8. Миграция с текущего сайта (fcarsenal-92.tilda.ws)

### 8.1. План перехода

1. **До запуска:** не трогать текущий сайт на Tilda
2. **В день запуска:**
   - Привязать домен fcarsenal92.ru к новому серверу (DNS A-запись)
   - Подождать распространения DNS (до 48 часов)
   - Убедиться что HTTPS работает (Let's Encrypt)
3. **После запуска:**
   - На Tilda-сайте добавить meta refresh / JavaScript redirect на fcarsenal92.ru
   - Или: если возможно, настроить 301 redirect с fcarsenal-92.tilda.ws на fcarsenal92.ru
   - Подать заявку на «Переезд сайта» в Яндекс.Вебмастер
   - В Google Search Console: добавить новый домен, запросить переиндексацию
4. **Через 3-6 месяцев:**
   - Проверить что весь трафик идёт на новый домен
   - Отключить Tilda-сайт

### 8.2. Сохранение позиций

- Текущий сайт не имеет SEO-позиций (нет H-тегов, nofollow, нет sitemap) -- потеря минимальна
- Основная ценность -- бренд-запросы «арсенал 92 севастополь» -- быстро перейдут на новый домен
- Новый сайт с правильным SEO начнёт ранжироваться в течение 2-4 недель для low-competition запросов
- Для средне-конкурентных запросов -- 2-4 месяца до устойчивых позиций
